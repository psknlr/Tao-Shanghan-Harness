# 能力成熟度與邊界聲明

本文檔採納外部評審的成熟度分級，作為**對外表述的規範**：A 級可放心使用，
B 級如實標注差距，C 級**禁止**在任何宣傳/論文/演示中聲稱。
（文檔數字類一致性由 `tests/test_docs_sync.py` 守衛。）

## A 級：已經比較可用

原文檢索 · 條文解釋 · 方證規則查詢 · 方劑鑒別 · 六經教學 · 劑量比與換算 ·
注家分歧圖譜 · 條文關係圖譜 · 引用核驗（CitationGuard/EvidenceBinder）·
患者端安全治理（工具面硬隔離+紅旗分診）· MCP/tool-call 集成 · Web API ·
引文模式識別與計量網絡（含 scope 一致性審計）· 誤引檢測 · 金標準抽樣評估閉環。

## B 級：已實現，仍需增強（差距如實列出）

| 功能 | 現狀 | 差距 |
|---|---|---|
| 多假設方證裁決 | 三態裁決+反證+禁忌+追問 | 臨床複雜症狀（多病共存/非典型表述）泛化不足 |
| 多智能體合議 | 規劃→專家→批評→綜合，證據綁定 | 專家本質是規則工具視角；「獨立 evidence packet + 獨立 critique」的真模型專家見路線圖 |
| 複雜任務編排 | 任務圖+作用域子代理 | local 路由對長複合句仍會漏拆（鑒別路由已於本輪增強簡繁與口語變體） |
| 深度研究循環 | 七維覆蓋+缺口驅動 | 屬工具編排型研究，非開放域自主科研發現 |
| 方解/藥解 | 方解四層口徑較完整 | 藥解以藥證事實統計為主；本草層已接原文摘錄+性味逐字提取+方中作用事件，但未做全本草結構化對齊 |
| 笈成全庫 | 檢索/閱讀/引文掃描/本草層 | 默認不下載（69MB 選裝）；全庫功能 UI 化僅工具台一入口 |
| 論文生成 | 模板+計量表格+增益層草稿 | 離可投稿仍需人工編輯與同行審閱 |

## 評測指標的性質（九輪評審採納：指標不得越級宣傳）

系統內各評測指標**分屬不同口徑**，引用時必須帶口徑，禁止混用：

| 指標 | 實測值 | 口徑 | 允許的表述 |
|---|---|---|---|
| 工具路由/軌跡合法/拒答精確（trajectory_eval） | 1.0 | **閉集規則回歸**（固定樣例、確定性後端）——是回歸護欄不是能力度量 | 「N 項閉集回歸樣例全通過」 |
| 故障注入恢復（perturbation_eval） | 1.0 | 閉集（3 場景注入） | 「注入場景下優雅降級」 |
| 規則內安全基準 | 100% | 閉集且樣本極小（9 題） | 只可作回歸護欄，不可作安全宣傳 |
| 遮方 LOCO / 獨立醫案回放 | 見 eval 產物（醫案回放 Top-1 遠低於規則內指標） | **開放泛化**——反映真實水平 | 引用時必須用這一組 |
| grounded_answer_rate | 1.0 | 引用合法性（編號真實+本輪取證） | 「引用合法」**≠**「結論成立」 |
| claim_grounding_rate | ~0.79（均值，逐題波動大） | 句級**詞彙重合下界**（lexical_overlap_v1，非語義蘊含） | 「句級詞彙級接地下界」 |

**禁止**：把閉集 1.0 指標宣傳為「智能體真實能力」「臨床可靠性」；
把 grounded_answer_rate 宣傳為「所有結論有證據」。對外引用能力數字
一律優先用開放泛化組（遮方/醫案回放），並附樣本量。

## 表述規範

- 「最早/首見」必須寫「在庫首現」或「宋本條文序首見」（見 `citable_statement`）；
- 引用計量結論須附 scope 與朝代範圍；
- 學派歸屬、方證觀點命題須標 posthoc_induction；
- 評測數字必須附口徑（閉集回歸 / 開放泛化 / 詞彙級下界，見上表）；
- 凡 C 級表述出現在任何材料中，按缺陷處理。
